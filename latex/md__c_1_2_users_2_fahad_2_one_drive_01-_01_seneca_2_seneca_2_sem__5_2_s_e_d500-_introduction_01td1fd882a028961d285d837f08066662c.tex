To use GLFW as a DLL, link against the {\ttfamily glfw3dll.\+lib} file for your environment. This will add a load time dependency on {\ttfamily glfw3.\+dll}. The remaining files in the same directory are not needed.

This DLL is built in release mode for the Multithreaded DLL runtime library.

There is also a GLFW DLL and import library pair in the {\ttfamily lib-\/static-\/ucrt} directory. To use GLFW as a static library, link against {\ttfamily glfw3.\+lib} if your application is using the Multithreaded DLL runtime library, or {\ttfamily glfw3\+\_\+mt.\+lib} if it is using the static Multithreaded runtime library. The remaining files in the same directory are not needed.

To use GLFW as a DLL, link against the {\ttfamily libglfw3dll.\+a} file for your environment. This will add a load time dependency on {\ttfamily glfw3.\+dll}. The remaining files in the same directory are not needed.

The DLLs are built in release mode.

The DLLs depend on the {\ttfamily msvcrt.\+dll} C runtime library. There is also a GLFW DLL and import library in the {\ttfamily lib-\/static-\/ucrt} directory that is built with Visual C++ 2019 and statically linked against the UCRT.

All DLLs in this archive provide the same ABI and can be used as drop-\/in replacements for one another, as long as the C runtime library they depend on is available.\hypertarget{md__c_1_2_users_2_fahad_2_one_drive_01-_01_seneca_2_seneca_2_sem__5_2_s_e_d500-_introduction_01td1fd882a028961d285d837f08066662c_autotoc_md6}{}\doxysubsection{\texorpdfstring{GLFW as a static library}{GLFW as a static library}}\label{md__c_1_2_users_2_fahad_2_one_drive_01-_01_seneca_2_seneca_2_sem__5_2_s_e_d500-_introduction_01td1fd882a028961d285d837f08066662c_autotoc_md6}
To use GLFW as a static library, link against the {\ttfamily libglfw3.\+a} file for your environment. The other files in the same directory are not needed.

The library is built in release mode and do not contain debug information.
